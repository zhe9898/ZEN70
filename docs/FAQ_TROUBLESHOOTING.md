## 🛑 避坑 1：装完 CPU 为什么疯狂满载？电脑会卡死吗？

**现象：**
刚点击完“一键部署”或系统刚拉起时，Windows 系统的任务管理器里，CPU 占用率飙升到 60% 甚至 100%。

**真相：**
这是**由于 Windows 自带的杀毒软件（Windows Defender，进程名为 `MsMpEng.exe`）在疯狂作祟**！
ZEN70 部署时会产生大量微小的配置文件读写，同时拉起 Docker (WSL2) 引擎。这属于高强度的合法 IO 行为。而 Windows 杀毒软件极度死心眼，会对这几万个小文件进行实时扫描拦截，活活拖累了 CPU 和部署速度。

**🌟 终极解决方案（必做）：把 ZEN70 放入杀毒软件白名单**
1. 按键盘 `Windows徽标键`，搜索并打开 **“Windows 安全中心”**（或“病毒和威胁防护”）。
2. 找到 **病毒和威胁防护** -> 点击蓝色的 **管理设置**。
3. 往下滚动到最底部的 **“排除项”** -> 点击 **添加或删除排除项**。
4. 点击 **[+添加排除项]** -> 选择 **[文件夹]**。
5. **请务必把你存放 `zen70` 代码的这个主文件夹全部选中添加！**
6. *(可选)*：如果你在网页里填了 `D:\ZEN70_Media` 作为影音盘，建议把这个 D 盘的文件夹也加入排除项。
👉 加入白名单后，CPU 的无谓消耗瞬间释放，你的电脑会立刻恢复丝滑平静！

## 🛑 避坑 2：未来家里设备多，全连在主控机会不会拖垮系统？（下沉分离）

**用户的担忧：**
“以后装了十几个温度计、各种摄像头、还要搞 AI 大模型推理，这台电脑吃得消吗？”

**ZEN70 的超前架构解答：完全吃得消，因为我们支持【极客级算力下沉部署】！**
目前你部署的这台电脑是**司令部（Master 主控中心）**，它只负责接收数据和发号施令。真正繁重的工作（脏活累活）绝对不会让它干！

1. **AI 算力（最费电脑的活）：独立显卡机**
   如果你以后想跑本地 AI（比如深度对话），并不需要让这台主控机受苦。你可以找另一台带独立显卡的电脑/挖矿机运行推理端，ZEN70 司令部只需要通过内网 IP 与它打个招呼就能使用它的算力。

2. **传感器网关（最乱最渣的活）：全扔给一百块钱的小树莓派**
   家里的蓝牙、Zigbee 信号，强烈建议**不要**直接连接主控机（极容易导致 USB 冲突）。
   你可以买个极其便宜的微型电脑在客厅跑一个小巧的**“边缘探针（Edge Node）”**。

**🚨 关键：需要我自己花几个月写下沉的代码吗？**
**绝对不用！我们承诺 0 代码全自动！**
在 ZEN70 未来完全体的控制面板里，你只需要点击【添加下沉节点】，在下拉菜单里选择你想要的组件（如：蓝牙网关节点）。
后台会自动为你生成专属于这个边缘小机器的一键安装代码（就像 Cloudflare Token 一样）。你只需在小机器上回车执行，它就会自己变身为哨兵，自动向主控中心实时汇报情报，彻底实现主控机的 **0 负载负担部署**！

## 🛑 避坑 3：在外网访问报错 1033 (Cloudflare 隧道错误)

如果你填了 Cloudflare Token，部署完去外网却看到极其刺眼的 `Error 1033` 网页，说明你**少做了一步至关重要的路由绑定**！

门卫虽然认识了你的电脑，但你没告诉门卫哪条管子通向 ZEN70。
**解决方案：**
立即打开本地的配置网页，点击密码框旁边的 **「不会填？看图文教程」** 按钮，或者查阅文件夹中的 `CLOUDFLARE_GUIDE.md`。
里面有详细说明：你需要回到 Cloudflare 后台的 Public Hostname 选卡里，把 `Service` 选为 `HTTP`，`URL` 硬性填入 `localhost:8000`。保存后等待 20 秒，1033 报错就能永远根除。
